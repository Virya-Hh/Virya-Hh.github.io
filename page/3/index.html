Java底层基础容易遗忘知识点总结

  1. jdk/jre的概念及相互之间的关系

    jvm=java虚拟机

    jre=jvm+核心类库 （jre是Java的运行环境）

    jdk=jre+开发工具 （JdkJava 语言的软件开发工具包）

  2. java语言诞生于1995年5月23日

  3. ‘\n’指的的是回车，就是另起一行;’\r’指的的是换行，就是回到一行的开头
  4. 布尔值 boolean 当声明一个变量时，占用4个字节，当声明一个数组时，每个数组元素占用一个字节。
  5. 负数的存储原理：-7 先求-7的绝对值，然后求其二进制，取反+1 即为-7的补码。
  6. % 正负号取决于被除数
  7. 编译器在编译的时候能确定常量的值，而不能确定变量的值，变量在运行时分配存储空间。但是+=在编译时可以进行强制类型转换，因此当用+=连接变量时，如果没有超过范围，是不会因为没有进行强制类型转换而报错的。
  8. switch-case
    1）. switch 后边只可能跟 byte char int double short
    2）. 从上往下执行，若匹配上某case，执行cese中的代码，若没有遇break，则会不断的往下执行，直到遇到break
    3）. 无论顺序如何，永远先判断case, 没有符合的才执行default

立体匹配算法总结及研究进展

A Taxonomy and Evaluation of Dense Two-Frame Stereo
Correspondence Algorithms

双目立体匹配算法的研究与进展

几种典型的立体匹配算法

立体匹配算法进展

根据匹配算法使用的约束信息的不同，立体匹配算法总体上分为局域算法、全局算法及半全局匹配算法三种。

基于局部约束的算法

局域算法利用的是对应点本身以及邻近的局部区域的约束信息，局域算法的优点是效率高，但是它对局部的一些由于遮挡和纹理单一等造成的模糊比较敏感，易造成误匹配。

优点：涉及信息量较少，计算复杂度低，实时性高； 
缺点：对噪声较敏感，对无纹理区域、视差不连续区域和遮挡区域匹配效果差；

分类：

  1. 区域匹配算法：

区域匹配直接利用图像的灰度信息，主要用于表面光滑以及具有明显纹理特征的图像，使用区域匹配可以直接获得稠密的深度图，但是对于缺乏纹理和深度不连续的情况，适应性较差，且这种方法的计算量很大，匹配精度较差。

图像配准 基于灰度的模板匹配算法（一）：MAD、SAD、SSD、MSD、NCC、SSDA、SATD算法

基于灰度的图像匹配方法研究

  1. 特征匹配算法：

特征匹配基于图像的几何特征，如边缘、轮廓、拐点、线段等对图像进行匹配，由于几何特征的稀疏性和不连续性，因此特征匹配只能得到稀疏的深度图，需要通过内插方法才能得到稠密的深度图，特征匹配以几何特征为基元，不易受光线的影响，因此鲁棒性较好，而且计算量小，速度快。

Java中对于字符串的认识

创建String类的对象的两种方式：

  1. “”直接赋值法

String Str = “jack”这个语句会先检查字符串常量池是否存放这个”jack1”这个字符串对象，如果没有存在，那么就会在字符串常量池中创建这个字符串对象，如果存在直接返回该字符串的内存地址值。

  1. new关键字法

String str3 = new String(“jack”) 该语句会创建两个对象,首先会先检查字符串常量池中存不存在jack这个字符串对象，如果不存在就会创建，如果存在就返回堆内存地址值。创建了出来之后，new String这个语句就会在堆内存中开辟一个字符串对象。总共两个对象。

补充：“==”用于比较引用数据类型的时候比较的是两个对象的内存地址；equals方法默认比较两个对象的内存地址

关系模型

主键：关系型数据库中的一条记录中有若干个属性，若其中某一个属性组(注意是组)能唯一标识一条记录，该属性组就可以成为一个主键。

外键：外键用于与另一张表的关联。是能确定另一张表记录的字段，用于保持数据的一致性。

绑定域名出现的问题及解决

Question： 404：There isn’t a GitHub Pages site here.If you’re trying to publish one, read the full documentation to learn how to set up GitHub Pages for your repository, organization, or user account.

Tinking:

1. 域名状态：注册宽限期（addPeriod）
2. DNS非万网DNS，检查是否有影响
3. 域名与github关联是否有问题
4. Ping+域名，每次查询IP地址都会发生变化，原因？

Solution:

  1. 实验证明，addPeriod状态时，可以正常访问网站。具体说明参考域名状态解析

  2. 本质上说说没有影响的，只是选择了不同的DNS服务器进行解析，但是用其他的DNS服务器，需要自己去检测时候可以解析，而使用万网的DNS服务器，会自动选择可以解析的服务器，因此最好选用万网DNS。此处的坑：修改DNS之后会提醒你去查看whios的DNS信息，若不一致，建议修改为一致的。这里需要时间等待，一般情况下修改完之后隔一段时间whios的会进行更新，不要一点开看到不一样就又修改为原来的DNS，这里浪费了很长时间。

  3. 不同的时间段，中间的ip地址会有一定的变化，因此在进行解析的时候不要添加A，而是添加CNAME，这样添加的值是网址而不是IP地址。

  1. 黑白盒测试的概念性问题
    黑盒测试 也称功能测试或者数据驱动测试或规格说明的测试，是从用户观点出发的测试。不考虑内部结构，只知道输入与输出之间的关系或者程序功能需求的情况下，来确定测试用例和推断测试结果的正确性和可操作性。黑盒测试方法主要有等价类划分法、边界值分析法、因果图法、错误推测法、综合策略、正交分析法，用于软件的确认测试。
    白盒测试也称逻辑驱动测试或者结构测试或基于结构的测试。它依赖于对程序细节的严密检验，针对特定条件和循环集设计测试用例，对软件的逻辑路径进行测试。在程序不同点检验程序的状态，来判断真实情况是否和预期的状态相一致。白盒测试方法主要包括逻辑覆盖发法，主要包括语句覆盖、判断覆盖、条件覆盖、判断条件覆盖、条件组合覆盖、路径覆盖
