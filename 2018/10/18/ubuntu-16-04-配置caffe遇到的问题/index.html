ubuntu 16.04 配置caffe遇到的问题

  • 科研

下载caffe安装包

git clone https://github.com/BVLC/caffe.git

Question:
下载文件并将本地文件复制到远程服务端

scp -r 目录名用户名 root@计算机ip 远程路径
由于权限问题失败了，尝试了下一种方法，即在anaconda环境下进行安装

在anaconda3环境下安装caffe

  1. conda install -c anaconda caffe-gpu

Question:由于网络连接问题，部分包没有下载成功

Solution:

(1) 查看下载的包的位置，卸载掉重新安装
通过 ls –full-time 来查看该路径下所有文件的最后修改时间进行判断

(2) 修改命令为：
conda install anaconda caffe-gpu
-c指的是在指定官网进行下载，可能由于网络或者其他原因无法获取url

linux 指令总结(2018/10/22)

  1. conda 安装指定版本的指定包

    conda config –add channels https://mirrors.tuna.tsinghua.edu.cn/anaconda/pkgs/free/

(使用清华镜像进行安装)

conda config --set show_channel_urls yes

  1. 安装或更新包（指定版本）

    conda install package=version

例如：
conda install numpy=1.9.3 conda update numpy=1.93

  1. nvcc -V 查看cuda版本
  2. ls -all pytorch.t ar 列出该文件的所有信息
  3. stat pytorch 查看文件大小
  4. Linux下各种解压方式
  5. vim README.md 文本编辑器可以打开md等格式的文件
  6. conda list 列出conda路径下的文件
  7. rm -f libstdc++ 强制删除libstdc++(-f 表示force 强制删除的意思)
  8. conda install anaconda hdf5=1.8.20(下载指定版本的hdf5,和上文的conda下载有一点不同，下来再查看)
  9. 用到的版本： cudnn 7.1.3 conda 8.0
  10. pytorch官网可以进行版本号的查询以及下载指令查询
  11. 清华镜像 /main/linux-64/

下载pytorch框架

conda install pytorch torchvision cuda80 -c pytorch

通过conda进行下载 使用的cuda 版本是8.0 其中-c可以去掉，-c的意思是在指定的channel进行下载

pip3 install http://download.pytorch.org/whl/cu80/torch-0.4.1-cp35-cp35m-linux_x86_64.whl

pip3 install torchvision

根据pytorch选择版本后，会出现相应的版本下载的指令，通过查看我们的是Stable–>Linux–>Pip–>Python 3.5–>CUDA8.0,其余的指令可以通过相应选择进行查看。

相关linux指令补充：

  1. nvidia -smi 查看GPU使用情况

NVDIA驱动下载

2.下载nvida
点击驱动程序下载，进行型号的选择

下载完成之后进行安装

i) dpkg -i nvidia-diag-driver-local-repo-ubuntu1604-384.145_1.0-1_amd64.debfor Ubuntu

ii) apt-get update

iii) apt-get install cuda-drivers

iv) reboot

补充： 如果之前有安装过别的NVDIA,但是需要换驱动的话：

  1. 若安装的其他的版本的话
    sudo apt-get remove –purge nvidia-331-updates

  2. 若安装的是官方版本的话
    运行run文件来卸载 sh ./nvidia.run --uninstall

CUDA 9.0下载

  1. 不要在清华镜像进行下载，清华镜像下载都是一个库函数不能进行安装
  2. 可以在cuda官网进行下载

Run sudo sh cuda_9.0.176_384.81_linux.run
